When you started this project, you made up your mind about including a raspberry pi and camera. You found good success in creating a network controlled wireless rover with a live camera feed. However, I should say that most of everything you accomplished here could have been done without the use of a raspberry pi. You mentioned that the controls were quite laggy, which is an inherent problem of exchanging data over a network. By either using a bluetooth capable game controller with the appropriate arduino shield, or an RC controller with the appropriate receiver, you would be able to achieve 0 latency control with multiple miles of range (if using RC hardware that runs ELRS). The use of a network for control here comes at the cost of range and latency, and a less than ideal control interface (your hardware is capable of it, but can you get granular control inputs with a keyboard?) I figure that the reason you wanted to use a raspberry pi and camera in the first place is to challenge yourself. So, after all I've said, I challenge you take it a step forward. Firstly, that raspberry pi is a powerful computer and right now most of that processing power is going unutilized. You should create an autonomous mode for the rover. Use the camera and raspberry pi to implement some basic image recognition and obstacle detection / avoidance. The goal is autonomy. You said the coding part of this project was easier than you expected, this will be a challenge. Secondly, and still in service of the goal of the autonomy, you can ditch the wifi communication in favor of cellular. There are cellular modules for raspberry pi available for sale, and having one would give you effectively infinite range. Next, incorporate a GPS / compass module into the rover, so that it can be aware of its own position at all times and be able autonomously complete waypoint missions. Lastly, and perhaps the most extra suggestion, is using an FPV grade video system in order to get a 0 latency video feed. Just look up "digital FPV kit" if you're curious (The caveat would be the video transmission will run on completely independent hardware to the rest of the rover. Alright, them's my thoughts, good work mate.

Great Project!  A few tricks on the DC-DC converters... For the servos, look into a BEC for an RC plane. Much smaller than the the DC-DC you used. For the Brushed DC motors, the voltage isn't the problem, it's the current. If you set the max current limit to below the max the motors can take, they can "Run at any voltage" (as the H-Bridge will protect them.) Also, you can just run the bridges off full voltage, but limit the max PWM to 50%, for half voltage. (The current limit would be another level of protection.)  Last, look into Conformal Coating for all your PCB boards to make them water proof-ish. Dialetric grease in all the connectors (LOTS OF IT) is your friend.
