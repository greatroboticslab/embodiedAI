Hi, Always remember that the code will always do what you tell it to.  In this case, see the loop with i as outer loop and the loop with j as inner loop.  Both the loops are starting from zero.  The flow is like this :  The flow enters into the outer loop and then into the inner loop. While the inner loop is running, the j variable is incrementing and nothing is happening to Variable i. Once the inner loop has finished executing, then the i Variable will increment by one and then again the flow will go into inner loop. And it will be reinitialised by 0 because it will be a fresh run for the inner loop as the previous run was over. But the outer loop is still going on. And it will go on till the condition specified becomes false.   For each iteration of outer loop, a complete inner loop will run.   I hope it cleared your doubt.

@allinallno.1934¬†  most of the motor drivers work on a set pattern. They have two control pins. If you give 0 (low) state to both the pins, the connected motors stays off. If you give 1 (high) state to both of them, then also, the connected motor stays off. If you give high state to one pin and low state to other pin, the motor starts moving. Let's say it's moving clockwise with respect to you. Now if you reverse the state of pins, the motor will move anti clockwise with respect to you.   Some other motor drivers may have an additional enable pin which you need to set high. Also note that you may need to use a level shifter as some motor drivers may not consider 3.3V as high state.
